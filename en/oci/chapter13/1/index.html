13.1 Preparation for Autoscaling

Create an instance for Autoscaling

Autoscaling requires a reference image. Let’s create an image with Apache installed on Oracle Linux used in the Load Balancer chapter so that one additional instance of Apache can be started for server extension through Autoscaling. Also consider using cloud-init, considering how to get the latest sources for additional instances to run. At this time, to make it easier to confirm that it is an additional instance, the private IP of the created host is visible in index.html.

  1. Go to the Compute Instance creation screen.

  2. Click Show advanced options below

  3. You can use cloud-init to set a script to start on instance creation. Here, we use a bash script to install Apache and create index.html.

    #!/bin/bash
sudo yum -y install httpd
sudo firewall-offline-cmd --add-port=80/tcp
sudo systemctl enable httpd
MY_IP=$(hostname -I)
sudo bash -c "echo 'Hello Apache on '$MY_IP >> /var/www/html/index.html"
sudo reboot

  4. Click Create below to create the instance.

  5. cloud-init log You can check the execution log of cloud-init in the
    /var/log/messages file.

Caution

If you use the firewall-cmd syntax used in the previous Load Balancer exercise in cloud-init as it is, the following error occurs during cloud-init. So I use the firewall-offline-cmd syntax instead.

Create Instance Configuration

Instance Configuration is required to create the configuration information of the instance created during autoscaling. make them in the following order

  1. Go to the detail screen of the instance you want to use.

  2. Click Create instance configuration in the additional action menu

  3. Enter the Instance Configuration name and click Create

  4. Complete creation

Create Instance Pool

Instance Pool is a collection of instances based on the same configuration created through Instance Configuration.

  1. To use a Load Balancer in the Instance Pool, you must create it in advance. Use the Load Balancer created in the previous chapter, or create one in advance if you don’t have one.

  2. Click Create instance pool in the Instance Configuration you created earlier.

  3. Basic Information on Instance Pool Creation

    • Name: Enter the desired name
    • Number of instances: the number of instances to start with pool creation, once set to 1

  4. Location information

    • Select the domain, VCN, and Subnet where the VMs to be created will be located.

  5. Load Balancer Settings

    • Check Attach a load balancer because Load Balancer is required for distribution between instances.
    • We use the one we created in the Load Balancer chapter. If not, create a new one.
    • Load balancer: Select the Load Balancer to use
    • Backend set: Select the Backend Set as the distribution target of the Load Balancer
    • Port: Enter the port to listen on
    • VNIC: Select as there is only Primary VNIC

  6. After confirming the selected settings, click Create.

  7. Instance Pool is created and 1 instance is created

  8. Go to Resources » Load balancer and click on the associated Load Balancer

  9. In LB details, go to Backend Set and click Backend Set used by Instance Pool

  10. You can see that it has been automatically added to the Backend Set. Also, you can see that the Health is OK and normal communication with the Apache server in the created VM.

  11. Connect to LB’s Public IP through a browser. You can see that it is configured normally.
